## Модель данных (store/types.ts + store)

Расширяю `SupportMessage`:
```
{
  id, sender: 'user'|'admin'|'bot',
  kind: 'text'|'image'|'file'|'system'|'quick_form',
  text, attachments?: [{name,url,mime,size}],
  created,
  read_by_admin?: boolean, read_by_user?: boolean,
  reply_to?: id,
  deleted_for?: 'user'|'all',  // 'user' = скрыто только у клиента, 'all' = удалено везде
  ticket_id?: string,
}
```
Новый стейт: `supportTickets: [{id, category, status: 'triage'|'open'|'closed', opened, closed?}]`, `adminPresence: { online: boolean, lastSeen: iso }`, `userTyping`, `adminTyping` (live, без фейка).

## Support.tsx (клиент-чат)

1. **Шапка**:
   - Аватарка — настоящий брендовый монограм Fanvue (gradient + буква F с градиентом + soft glow), не «иишная».
   - Имя — `Fanvue Care` (RU: «Fanvue · Забота»).
   - Под именем: реальный статус оператора. Зелёная точка только если `adminPresence.online`. Иначе «был(а) недавно / в этом часу / 2 часа назад / сегодня в 14:30 / вчера / на этой неделе» исходя из `lastSeen`.
   - Кнопка `i` справа открывает bottom-sheet с: время работы, как составить заявку, частые вопросы (раскрывающиеся), ссылка на Telegram.

2. **Триаж-бот** (стартовое сообщение):
   - При пустом чате (или после `closed` тикета) бот присылает сообщение + 3 кнопки-категории: Платёж / Доставка / Связь с оператором.
   - Выбор → бот задаёт уточняющие вопросы → собирает заявку (`quick_form`) → создаёт ticket со статусом `open` → дальше обычный чат с админом.
   - История тикетов сохраняется и показывается в чате (system-плашки «Заявка #FV-1234 · открыта/закрыта»).

3. **Composer**:
   - Слева кнопка скрепки → `<input type="file" accept="image/*,application/pdf,...">`. Превью прикреплённых файлов над инпутом, кнопка удалить превью.
   - Картинки кодируются как dataURL и кладутся в `attachments` (mock без бэкенда; `tgNotify` уведомляет админа).
   - Long-press / swipe-right на сообщение → меню «Ответить / Удалить». Reply показывает цитату над инпутом + якорь в бабле.

4. **Удаление**:
   - User: "удалить у себя" → `deleted_for: 'user'`, у клиента место занимает плашка «Сообщение удалено · посмотреть» (только у админа в админке тоже видно через раскрытие).
   - Сообщения админа клиент удалить не может.

5. **Read receipts**: при заходе в чат + при появлении нового сообщения админа отмечаем `read_by_user=true`. Свои сообщения показывают ✓ (доставлено) → ✓✓ (прочитано админом, когда `read_by_admin`).

6. **Анимации**: spring entry для бабблов, layout-анимация при reply/удалении, мягкий heartbeat у online-точки, плавный typing-индикатор только при реальном вводе. Убрать фейковый авто-typing.
